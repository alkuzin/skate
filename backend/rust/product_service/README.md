# Микросервис управления товарами

## Описание

Данный микросервис, реализованный на Rust с использованием фреймворка Actix Web,
отвечает за управление ресурсами, связанными с товарами и их категориями.

Микросервис управления товарами предоставляет следующие маршруты для работы с
товарами и их категориями:
- **Создание товара `(POST /products)`**: Позволяет клиентам создавать новые
товары, отправляя данные о товаре в теле запроса.
- **Создание категории `(POST /categories)`**: Позволяет клиентам создавать
новые категории товаров, отправляя данные о категории в теле запроса.
- **Получение товара `(GET /products/{id})`**: Позволяет клиентам получать
информацию о конкретном товаре по его идентификатору.
- **Получение категории `(GET /categories/{id})`**: Позволяет клиентам получать
информацию о конкретной категории по её идентификатору.
- **Обновление товара `(PUT /products/{id})`**: Позволяет клиентам обновлять
существующий товар, отправляя обновленные данные о товаре в теле запроса.
- **Обновление категории `(PUT /categories/{id})`**: Позволяет клиентам
обновлять существующую категорию, отправляя обновленные данные о категории в
теле запроса.
- **Удаление товара `(DELETE /products/{id})`**: Позволяет клиентам удалять
товар по его идентификатору.
- **Удаление категории `(DELETE /categories/{id})`**: Позволяет клиентам
удалять категорию по её идентификатору.
- **Получение списка товаров `(GET /products)`**: Позволяет клиентам получать
список всех товаров, хранящихся в базе данных.
- **Получение списка категорий `(GET /categories)`**: Позволяет клиентам
получать список всех категорий товаров, хранящихся в базе данных.
- **Получение товаров по категории `(GET /categories/{id}/products)`**:
Позволяет клиентам получать список товаров, относящихся к конкретной
категории по её идентификатору.

## Структура

- `Cargo.lock` - Этот файл автоматически генерируется и содержит зафиксированные
версии всех зависимостей микросервиса. Он гарантирует, что при сборке проекта на
разных машинах будут использоваться одни и те же версии библиотек, что помогает
избежать проблем с совместимостью.
- `Cargo.toml` - Это основной файл конфигурации микросервиса. Он содержит список
зависимостей, необходимых для сборки и работы программы.
- `src` - Директория, содержащая исходный код микросервиса.
  - `category_repository.rs` - Реализация структуры, отвечающей за
  взаимодействие с базой данных для категорий товаров.
  - `config.rs` - Содержит настройки конфигурации микросервиса.
  - `lib.rs` - Файл содержит публичные API микросервиса.
  - `main.rs` - Точка входа программы.
  - `product.rs` - Файл содержит структуры и перечисления, связанные с заказами
  и их категориями.
  - `product_repository.rs` - Реализация структуры, отвечающей за
  взаимодействие с базой данных для товаров.
  - `service.rs` - Реализация главной структуры микросервиса.
- `tests` - Директория, содержащая тесты программы.
  - `integration_test.rs` - Содержит интеграционные тесты, которые проверяют
работу публичных API микросервиса.

## Установка

Для сборки микросервиса нужно запустить следующую команду:
```shell
cargo run
```
Для того чтобы вернуть проект в изначальное состояние, нужно запустить
следующую команду:
```shell
cargo clean
```

## Тесты

Для запуска unit и интеграционных тестов нужно запустить следующую команду:
```shell
cargo test
```
Для запуска unit и интеграционных тестов с отладочным выводом нужно запустить
следующую команду:
```shell
cargo test -- --nocapture
```

## Документация

Для сборки документации микросервиса нужно запустить следующую команду:
```shell
cargo doc
```

Для просмотра документации микросервиса в браузере нужно запустить следующую
команду:
```shell
cargo doc --open
```
